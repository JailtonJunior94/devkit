# Using References

Templates are the primary tool for writing READMEs. References provide depth and should be loaded only when they are needed for the current project or edge case.

## Reference Selection

### art-of-readme.md
`references/art-of-readme.md`

Why: Explains the philosophy behind great READMEs and how readers actually scan and evaluate projects.
What: Covers cognitive funneling, brevity as a feature, and README content that keeps users out of source code when possible.

### make-a-readme.md
`references/make-a-readme.md`

Why: Provides practical section-by-section guidance for what to include.
What: Walks through common sections such as name, description, installation, and usage with concrete suggestions.

### standard-readme-spec.md
`references/standard-readme-spec.md`

Why: Helps when consistency or formal compliance matters.
What: Describes required versus optional sections, recommended ordering, and formatting expectations.

### standard-readme-example-minimal.md
`references/standard-readme-example-minimal.md`

Why: Shows a minimal compliant example.
What: Demonstrates the leanest README structure that still communicates the essentials.

### standard-readme-example-maximal.md
`references/standard-readme-example-maximal.md`

Why: Shows a full-featured example.
What: Demonstrates a richer README with badges, table of contents, and optional sections.
