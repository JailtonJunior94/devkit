# README Style Guide

## Common Mistakes

- **No install steps** - Never assume setup is obvious
- **No examples** - Show, don't just tell
- **Wall of text** - Use headers, tables, lists
- **Stale content** - Add "last reviewed" date
- **Generic tone** - Write for the actual audience

## Prose Quality

For general writing advice, clear prose, and AI patterns to avoid, use a dedicated writing skill when one is available.
